My First HTML Page

HTML May 15, 2024

I still remember the mix of excitement and nervousness I felt when I created my very first HTML page. It was like opening a door to a whole new world of possibilities. In this post, I'll share my journey, the mistakes I made, and what I learned from this foundational experience.

The Beginning

It all started with a simple text editor (I used VS Code) and a dream to build something visible on the web. My goal was modest: create a page about my favorite books that I could share with friends.

The Structure

I began with the basic HTML skeleton that every tutorial mentions:

<!DOCTYPE html>
     <html lang="en">
     <head>
    <meta charset="UTF-8">
    <title>My Favorite Books</title>
      </head>
      <body>
    <h1>Welcome to My Book Page</h1>
    <p>Here are some books I love:</p>
    <ul>
        <li>The Pragmatic Programmer</li>
        <li>Clean Code</li>
        <li>HTML and CSS: Design and Build Websites</li>
    </ul>
</body>
</html>

What I Learned

Creating this simple page taught me several fundamental concepts:

  1. Document Structure: How HTML documents are organized with head and body sections
  2. Basic Tags: The purpose of headings (<h1>), paragraphs (<p>), and lists (<ul>, <li>)
  3. Semantics: How different tags convey meaning about the content they contain
  4. File Saving: The importance of saving files with the .html extension
  5. Browser Rendering: How browsers interpret and display HTML

Tips for Beginners

  • Always start with the proper DOCTYPE declaration
  • Use semantic HTML elements where possible
  • Validate your code using the W3C Validator
  • Don't worry about styling at first - focus on structure
  • View your page in multiple browsers to check consistency

The Challenges

Of course, it wasn't all smooth sailing. I encountered several hurdles:

Common Beginner Mistakes I Made

  • Forgetting to close tags (especially nested ones)
  • Using spaces in filenames (caused loading issues)
  • Not understanding the difference between relative and absolute paths
  • Overusing <div> tags before learning semantic elements
  • Getting frustrated when my page didn't look "pretty" without CSS

The Final Result

After several iterations (and many browser refreshes), I ended up with a simple but functional page. It wasn't fancy, but seeing my content rendered in a browser for the first time was incredibly rewarding.

Reflections

Looking back, this simple exercise taught me more than just HTML syntax. It gave me:

  • Confidence that I could actually build something
  • An understanding of how web pages are constructed
  • The realization that mistakes are part of the learning process
  • Excitement to learn more (CSS was my next target)

If you're just starting your HTML journey, embrace the process. Every expert was once a beginner staring at their first <html> tag with a mix of wonder and confusion.
